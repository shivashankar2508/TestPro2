⚡ Quick Start

✅ Good News! OAuth is already integrated into TestTrack Pro. Just follow these steps to enable it.

Automated Setup (Recommended)

Run the setup script to configure your credentials:

python setup_oauth.py

The script will guide you through the entire process step-by-step!

🔧 Manual Setup

Step 1: Google OAuth Configuration

  1. Go to Google Cloud Console
  2. Create a new project named "TestTrack Pro"
  3. Enable "Google+ API" in the API library
  4. Click Credentials → + Create Credentials → OAuth 2.0 Client ID
  5. Select Web application
  6. Add Authorized Redirect URI: http://localhost:3000/auth/oauth-callback
  7. Copy your Client ID and Client Secret

Step 2: GitHub OAuth Configuration

  1. Go to GitHub Developer Settings
  2. Click OAuth Apps
  3. Click New OAuth App
  4. Fill in the form:
    Application name: TestTrack Pro
    Homepage URL: http://localhost:3000
    Authorization callback URL: http://localhost:3000/auth/oauth-callback
  5. Copy your Client ID and Client Secret

Step 3: Configure Environment

Create or update backend/.env file:

# Google OAuth GOOGLE_CLIENT_ID=your_google_client_id_here.apps.googleusercontent.com GOOGLE_CLIENT_SECRET=your_google_client_secret_here # GitHub OAuth GITHUB_CLIENT_ID=your_github_client_id_here GITHUB_CLIENT_SECRET=your_github_client_secret_here

Step 4: Restart Backend

Restart the backend server to load the new credentials:

cd backend $env:PYTHONPATH="$pwd" python -m uvicorn app.main:app --host 0.0.0.0 --port 8001 --reload

⚠️ Troubleshooting

Problem Solution
Redirect URI mismatch Ensure callback URL in Google/GitHub settings matches exactly: http://localhost:3000/auth/oauth-callback
OAuth buttons don't appear Clear browser cache (Ctrl+Shift+Delete), hard refresh (Ctrl+F5), restart backend
Invalid or missing credentials Verify Client ID/Secret are correct in .env file, restart backend
Stuck on login page Check browser console (F12) for errors, check backend logs
CORS error Verify ALLOWED_ORIGINS in .env includes http://localhost:3000

📡 API Endpoints

Method Endpoint Description
GET /api/auth/oauth/google/authorize Get Google authorization URL
GET /api/auth/oauth/github/authorize Get GitHub authorization URL
POST /api/auth/oauth/google/callback Handle Google OAuth callback
POST /api/auth/oauth/github/callback Handle GitHub OAuth callback
POST /api/auth/oauth/link Link OAuth account to existing account

🚀 Production Deployment

⚠️ Before deploying to production:
  1. Update callback URLs in Google/GitHub settings to your production domain
  2. Update FRONTEND_URL in .env to your production domain
  3. Update ALLOWED_ORIGINS to your production domain
  4. Set ENV=production
  5. Use a strong SECRET_KEY (32+ characters)
  6. Switch to PostgreSQL (not SQLite)
  7. Enable HTTPS only
